Vier gewinnt - Kurzdokumentation

Team: Myriam Schaad und Colin Dubuis

Funktionsübersicht

Die Anwendung bietet folgende Kernfunktionen, die durch den Custom Hook useConnectFour gesteuert werden:

  • Spielzug (Drop Piece): Durch Klicken auf eine Spalte fällt der Spielstein ("Gravity Logic") auf den tiefstmöglichen freien Platz. Der Zug wechselt automatisch zwischen Rot und Blau.
  • Gewinnprüfung (Win Check): Nach jedem Zug prüft der Algorithmus automatisch in vier Richtungen, ob vier gleiche Steine verbunden sind:
    • Horizontal
    • Vertikal
    • Diagonal (unten-rechts)
    • Diagonal (oben-rechts)
  • Undo-Funktion: Ermöglicht das Rückgängigmachen des letzten Zuges. Der Spielstatus wird auf den vorherigen Zustand aus der History zurückgesetzt.
  • Speichern & Laden (Persistence): Der aktuelle Spielstand inklusive Historie kann im localStorage des Browsers gespeichert und später wieder geladen werden.
  • Reset (Neues Spiel): Setzt das Spielfeld und die Historie auf den Anfangszustand zurück.

Wichtige Implementierungsdetails

Technologie-Stack

Das Projekt wurde mit folgenden Technologien umgesetzt:

  • Framework: React (v19)
  • Build Tool: Vite
  • Sprache: JavaScript (ES6+) / JSX
  • Styling: CSS3 (Grid Layout & Flexbox)

Architektur & Code-Struktur

Die Anwendung folgt dem Prinzip der "Separation of Concerns" (Trennung der Zuständigkeiten):

  • View (ConnectFour.jsx): Verantwortlich für die Darstellung der Benutzeroberfläche. Das Board wird mittels CSS Grid (7 Spalten x 6 Zeilen) gerendert. Es enthält keine Spielelogik, sondern konsumiert diese nur.
  • Logic (useConnectFour.js): Ein Custom Hook, der den kompletten useState für das Board, den aktuellen Spieler und den Spielstatus (Game Over) verwaltet.
  • Datenhaltung: Das Spielfeld wird als 2D-Array (6x7) repräsentiert. Ein History-Array speichert vergangene Zustände für die Undo-Funktion.
  • Storage Service: Ein integrierter Service kapselt die Zugriffe auf den localStorage, um Speicher- und Ladefehler abzufangen (Error Handling).

Bedienung

  • Klick auf Spalte um einen Stein zu setzen.
  • Undo entfernt letzten Zug (falls vorhanden).
  • Save speichert den aktuellen Stand und Load lädt den zuletzt gespeicherten Zustand.
  • Link zur Doku öffnet diese Seite in einem neuen Tab.

Besonderheiten

  • Animationen: CSS Keyframes (@keyframes drop) sorgen für eine flüssige Animation beim Einwerfen der Steine.
  • Responsivität: Das Layout passt sich mittels Media Queries (max-width: 800px) an mobile Endgeräte an, wobei die Anordnung der Steuerelemente und die Größe des Boards optimiert werden.
